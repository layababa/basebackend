## 可程序化扩展的高风险模式

实际注册拦截中，建议不要只依赖静态表。可以把以下模式加入生成式 denylist 或正则策略：

```text
^(\d)\1{3,}$                         # 同一数字重复，如 111111、88888888
^(?:0123456789|1234567890|9876543210).*$ # 数字连续/倒序前缀
^(?:qwerty|asdf|zxcv|qazwsx).*$       # 键盘连续串
^(?:19|20)\d{2}(?:0[1-9]|1[0-2])(?:0[1-9]|[12]\d|3[01])$ # 日期串
^(?:admin|root|test|user|guest|mima|qq|wx|weixin|taobao|baidu).{0,4}\d{1,8}$ # 常见词+数字
^(?:woaini|love|iloveyou|nihao|china)\d{0,8}$ # 常见拼音/英文情感词+数字
```
